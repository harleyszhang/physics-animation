```bash
请为我创建一个**面向中学生的物理学习网站**，要求包含**知识大纲**、**所有物理原理的可视化展示**、**可视化的物理知识学习路径**以及**示例演示**。请输出一个完整的 HTML 文件，包含内嵌 CSS 和 JavaScript，确保现代、美观、响应式，并能在浏览器中直接运行。
## 一、网站布局与整体风格
- **布局**：左侧固定宽度 240px 的知识大纲面板；右侧分为上下两部分：上方为可视化展示区（含 canvas 和交互滑块），下方为可视化学习路径图。
- **配色**：柔和清爽，以浅蓝、白色为主，点缀暖色（如橙色）表示交互元素。使用圆角卡片、轻微阴影，字体清晰（建议 Google Fonts 的 'Poppins' 或 'Roboto'）。
- **图标**：使用 Font Awesome 6 免费版 CDN 添加箭头、指示图标等。
## 二、知识大纲（左侧面板）
- 以树形/列表形式展示物理学主要分支和具体原理。至少包含：
  - **力学**：牛顿第二定律、浮力、杠杆原理
  - **光学**：光的反射定律、光的折射定律
  - **电学**：欧姆定律、简单串联电路
- 每个原理作为可点击项，点击后：
  1. 右侧可视化区域更新为该原理的交互式 canvas 绘图；
  2. 下方显示对应的原理解释和公式；
  3. 高亮当前选中的大纲项。
- 大纲可适当分组（如使用缩进），并带有简单的展开/折叠效果（默认全部展开即可）。
## 三、可视化展示区（右侧上方）
- 包含一个 **canvas 元素**（尺寸建议宽 100%，高 280px~320px），用于绘制当前原理的示意图。
- 每个原理应配备**交互滑块**（或输入框），让用户调节关键物理量（如力、质量、角度、电阻等），滑块变化时 canvas 实时重绘，并更新显示的数值。
- 下方显示该原理的**简短说明**（如公式、现象）。
### 必须实现的原理及绘图要求（至少 16 个以上）：
1. **牛顿第二定律 (F = ma)**
   - 绘制一个小车或方块，受向右的拉力，标注质量 m、力 F，显示计算出的加速度 a。
   - 滑块：力 F (1-10 N)、质量 m (1-5 kg)；显示 a = F/m。
2. **浮力 (阿基米德原理)**
   - 绘制一个浸入水中的立方体，画出浮力向上箭头，标注排开液体体积。
   - 滑块：物体密度 (0.5-2 g/cm³)，显示浮力大小及物体沉浮状态。
3. **杠杆原理**
   - 绘制杠杆、支点、左右两侧的力臂和力（可用箭头表示），标注力臂长度。
   - 滑块：左侧力 (1-10 N)、左侧力臂 (1-5 m)；右侧力臂固定 3 m，显示平衡时右侧需要的力。
4. **光的反射定律**
   - 绘制水平镜面、法线（虚线）、入射光线和反射光线，用弧线标注入射角和反射角。
   - 滑块：入射角 (0°~80°)，显示反射角 = 入射角。
5. **光的折射定律 (斯涅尔定律)**
   - 绘制两种介质（空气/水）界面，画出折射光线，标注入射角和折射角。
   - 滑块：入射角 (0°~80°)，假设折射率 n1=1.0, n2=1.33，显示折射角。
6. **欧姆定律 (U = IR)**
   - 绘制简单电路：电池、电阻、电流表，用线条连接。
   - 滑块：电压 (1-12 V)、电阻 (1-100 Ω)，显示电流 I = U/R。
等等
## 四、可视化学习路径（右侧下方）
- 设计一个水平/垂直的**路径图**，展示建议的物理知识学习顺序。例如：
  `力学基础 → 能量与功 → 振动与波 → 几何光学 → 电路入门`
- 每个节点设计为圆形或卡片，包含名称，节点间用箭头（→ 或 FontAwesome 图标 fa-arrow-right）连接。
- 点击任意节点，应能自动切换到对应的原理可视化（例如点击“力学基础”跳转到牛顿第二定律，点击“几何光学”跳转到光的反射/折射）。如果没有完全匹配的原理，可跳转到最接近的一个，或弹出提示。
- 路径图应具有视觉吸引力，背景柔和，节点悬停有效果。
## 五、示例演示
- 页面加载时，默认选中 **牛顿第二定律** 作为示例，可视化区显示其绘图和滑块，学习路径中对应的节点（如“力学基础”）高亮。
- 可额外在可视化区右上角添加一个“随机示例”按钮，点击后随机切换到一个原理，帮助用户探索。
## 六、技术实现要求
1. **纯 HTML/CSS/JavaScript**，不使用外部库（除了 Font Awesome 和 Google Fonts）。所有代码放在一个文件中，自包含。
2. **JavaScript 结构清晰**：
   - 使用一个 `principles` 数组存储每个原理的元数据：id、名称、分类、描述、绘图函数、滑块配置。
   - 绘图函数签名：`draw(canvasCtx, width, height, params)`，其中 params 为当前滑块值对象。
   - 切换原理时，销毁旧滑块并创建新滑块，绑定输入事件重新调用绘图函数。
3. **canvas 绘图**应使用基本绘图 API，确保在不同尺寸下自适应（可根据 canvas 实际宽高计算坐标比例）。
4. **响应式**：在屏幕宽度小于 700px 时，左侧大纲自动堆叠到上方，右侧上下区域保持。
5. **代码注释**：关键部分添加注释，方便理解。

- 不要遗漏 Font Awesome 和 Google Font 的 CDN 链接。
- 确保所有交互流畅，没有控制台报错。
- 对于未完全实现的原理（如折射），可用简单的示意绘图，但要传递核心概念。
- 尽量让界面生动有趣，适合中学生。
```
